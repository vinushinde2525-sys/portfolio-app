## 📖 About the Project

> **Vinayak Portfolio** is a production-grade full-stack MERN developer portfolio featuring an **immersive cosmic Three.js background**, animated planets, shooting stars, and glassmorphism UI — built with **React 18, Node.js, Express, and MongoDB** with a full JWT-protected admin panel.

Most developer portfolios are static HTML templates. This one is a **complete full-stack application** — with a live admin CMS, automated test suite, GitHub Actions CI/CD pipeline, and a premium cosmic visual experience that runs entirely in the browser using React Three Fiber.

This project demonstrates **senior-level frontend architecture**, scalable REST API design, production DevOps practices, and modern UI/UX — all deployed and live on Render.

## 🏗️ Architecture

```
Browser / Client
       │
       ▼
React SPA (Vite + React 18)
  ├── Framer Motion     →  Page transitions, scroll reveals, float animations
  ├── React Three Fiber →  Cosmic stars, planets, shooting stars (fixed bg)
  ├── TanStack Query    →  Server state: projects, skills, blogs
  ├── Zustand           →  Client state: admin auth token
  └── Axios             →  JWT interceptors, auto-redirect on 401
       │
       ▼  REST API
Express.js Server
  ├── Helmet            →  Secure HTTP headers
  ├── CORS              →  Allowlist-based origin validation
  ├── Rate Limiting     →  200 req/15min global · 10 req/15min auth
  ├── Mongo Sanitize    →  NoSQL injection prevention
  ├── JWT Middleware    →  Protect admin routes
  └── Morgan            →  Production + dev logging
       │
       ▼
MongoDB Atlas (Mongoose)
  ├── Admin    (email, hashed password, role)
  ├── Project  (title, tags, github, live, featured, image)
  ├── Skill    (name, category, level)
  ├── Blog     (title, content, published)
  ├── Message  (name, email, subject, message, read)
  └── Visitor  (page, ip, timestamp — analytics)
```

## ⚙️ Installation

**Step 1 — Clone**

```bash
git clone https://github.com/vinushinde2525-sys/portfolio.git
cd portfolio/portfolio-app
```

**Step 2 — Install**

```bash
# Backend
cd backend && npm install

# Frontend
cd ../frontend && npm install
```

**Step 3 — Configure environment**

```bash
# Backend — copy and fill in required values
cp backend/.env.example backend/.env
# Required: MONGO_URI, JWT_SECRET, JWT_REFRESH_SECRET
# Optional: Cloudinary, Email (SMTP), ALLOWED_ORIGINS

# Frontend
cp frontend/.env.example frontend/.env
# Set VITE_API_URL=http://localhost:5000
```

**Step 4 — Seed + Run**

```bash
# Seed database (from /backend)
npm run seed
# Prints admin credentials on success

# Run backend (from /backend)
npm run dev     # http://localhost:5000

# Run frontend (from /frontend)
npm run dev     # http://localhost:5173
```

**Admin panel:** `http://localhost:5173/admin/login`
Credentials printed by `npm run seed`.

## ⚡ Architecture Decisions

| Decision | Reason |
|---|---|
| TanStack Query for API data | Automatic caching, background refetch, deduplication — no manual loading state |
| Zustand for admin auth | Lightweight — no Redux boilerplate needed for a single auth token |
| R3F fixed canvas behind all pages | One Three.js context for the entire app — no re-init on route change |
| Framer Motion `AnimatePresence` | Enables exit animations during route transitions |
| Vite manual chunks | Splits Three.js (~800KB), Framer Motion into separate cache-busted bundles |
| JWT in-memory + httpOnly cookie | Access token in memory (no XSS risk), refresh token in httpOnly cookie |
| `forwardRef` on SkeletonCard | Required for Framer Motion `AnimatePresence` to ref the exiting element |
| MSW for frontend tests | API mocking at the network layer — tests are realistic without a real backend |

## 🧩 Challenges & Solutions

| Challenge | Solution |
|-----------|----------|
| Three.js canvas causing layout shifts | Fixed position behind app at `z-index: 0`, `pointer-events: none` |
| Framer Motion `PopChild` ref warning | Wrapped `SkeletonCard` in `React.forwardRef` |
| `localStorage` experimental warning in Vitest | Full `localStorage` mock in `setup.js` |
| Enum mismatch in seed data | Aligned `'Tools & DevOps'` to valid schema values `'Tools'` / `'DevOps'` |
| `seed.js` not found | Script was in `utils/` — created proper `scripts/seed.js`, updated `package.json` |
| `app.js` import errors | Route filenames were `authRoutes.js` but imports said `auth.routes.js` — fixed all 7 |
| React Router v7 warnings | Added `future={{ v7_startTransition, v7_relativeSplatPath }}` to `BrowserRouter` |
